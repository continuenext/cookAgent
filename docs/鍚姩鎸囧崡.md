# GustoBot 启动指南

## 前置条件

- macOS（Apple Silicon）
- Python ≥ 3.12
- Docker Desktop 已安装并运行
- 包管理器 [uv](https://docs.astral.sh/uv/)（`brew install uv`）
- 项目根目录已配置 `.env` 文件

---

## 第一步：启动基础设施

```bash
cd /path/to/gustobot
docker compose up -d
```

此命令会启动全部基础设施和可视化服务。

| 服务 | 主机端口 | 用途 |
|------|---------|------|
| neo4j | 17687 | 知识图谱（菜谱概念、食材关系、烹饪步骤） |
| mysql | 13306 | 关系数据库（菜谱元数据、统计数据） |
| redis | 6379 | LangGraph 会话检查点 + 会话元数据缓存 |
| etcd | — | Milvus 分布式协调（内部） |
| minio | 9000 / 9001 | Milvus 对象存储（内部） |
| milvus | 19530 | 向量数据库（菜谱文本语义检索） |
| mysql-adminer | 18080 | MySQL 可视化管理（Adminer） |
| redis (内置 RedisInsight) | 18081 | Redis 可视化管理（RedisInsight） |
| milvus-attu | 18000 | Milvus 可视化管理（Attu） |

查看启动状态：

```bash
docker compose ps
```

访问地址：
- Neo4j Browser: `http://localhost:17474`
- MySQL Adminer: `http://localhost:18080`
        - System: `MySQL`
        - Server: `mysql`
        - Username: `recipe_user`
        - Password: `recipepass`
        - Database: `recipe_db`
- Redis Insight: `http://localhost:18081`
        - 添加连接：Host=`redis`, Port=`6379`
- Milvus Attu: `http://localhost:18000`
        - Address: `milvus:19530`

> **注意**：Milvus 启动较慢（约 90 秒）。用 `docker compose ps` 确认所有容器状态为 `running`。

---

## 第二步：安装 Python 依赖

```bash
uv sync
```

此命令会读取 `pyproject.toml`，创建/同步 `.venv` 虚拟环境并安装所有依赖。

---

## 第三步：配置环境变量

在项目根目录创建 `.env` 文件（如果不存在），至少包含以下必填项：

```bash
# ── LLM（必填）──
LLM_API_KEY=你的阿里云百炼API密钥
LLM_MODEL=qwen3-max
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1

# ── Embedding（必填）──
EMBEDDING_API_KEY=同上
EMBEDDING_MODEL=text-embedding-v3

# ── Reranker（必填）──
RERANK_API_KEY=同上
RERANK_MODEL=qwen3-rerank

# ── 本地开发数据库连接 ──
DATABASE_URL=mysql+pymysql://recipe_user:recipepass@localhost:13306/recipe_db
NEO4J_URI=bolt://localhost:17687
REDIS_URL=redis://localhost:6379/0
MILVUS_HOST=localhost
MILVUS_PORT=19530
```

可选配置（按需添加）：
```bash
# 视觉模型（图片识别功能）
VISION_API_KEY=你的视觉API密钥
VISION_MODEL=glm-4.5-flash

# 图片生成
IMAGE_GENERATION_API_KEY=你的图片生成API密钥

# 调试模式（开启后支持热重载）
DEBUG=true
```

## 第四步：初始化数据

MySQL 容器首次创建时会自动执行以下 SQL 文件：

- `gustobot/data/init_mysql.sql`
- `gustobot/data/insert_sample_data.sql`

Neo4j 需要手动导入图谱数据。首次初始化执行：

```bash
docker compose exec neo4j cypher-shell -f /import/neo4j_import.cypher
```

验证 Neo4j 导入数量：

```bash
docker compose exec neo4j cypher-shell "MATCH (n) RETURN count(n) AS nodes; MATCH ()-[r]->() RETURN count(r) AS relationships;"
```

如果需要重新导入 Neo4j，请先清空旧图谱：

```bash
docker compose exec neo4j cypher-shell "MATCH (n) DETACH DELETE n;"
docker compose exec neo4j cypher-shell -f /import/neo4j_import.cypher
```

Milvus 向量索引依赖 Neo4j 数据和 Embedding API。Neo4j 导入完成后执行：

```bash
python -m gustobot.infrastructure.persistence.rebuild_milvus_index
```

验证 Milvus 集合：

```bash
python -c "from pymilvus import connections, utility, Collection; connections.connect(alias='default', host='127.0.0.1', port='19530'); print(utility.list_collections()); [print(name, Collection(name).num_entities) for name in utility.list_collections()]"
```

---

## 第五步：启动应用

```bash
python main.py
```

启动过程：
1. FastAPI lifespan 触发 → 初始化 LangGraph 主图（构建工作流 + 连接 Redis Checkpointer）
2. 注册 4 个路由模块：`/knowledge`、`/sessions`、`/chat`、`/upload`
3. Uvicorn 监听 `0.0.0.0:8000`

## 启动依赖链

```
docker compose up -d
        ↓
基础设施服务就绪（neo4j + mysql + redis + etcd + minio + milvus）
        ↓
Neo4j 导入图谱数据
        ↓
Milvus 重建向量索引
        ↓
python main.py
        ↓
lifespan() → init_graph() → LangGraph 工作流 + Redis Checkpointer 初始化
        ↓
FastAPI ready → http://0.0.0.0:8000
```

## 常见问题

### Milvus 启动失败
检查 etcd 和 minio 是否正常：`docker compose logs milvus`。Milvus 依赖这两个服务先就绪。

### Milvus 数据保存在哪里？每次重启要重建吗？
- Milvus 数据持久化在 Docker Volume 中：
        - `milvus_data`（Milvus 本体数据）
        - `etcd_data`（元数据）
        - `minio_data`（对象存储）
- 仅重启应用（`python main.py`）不会丢失向量数据，不需要重建。
- 仅重启容器（`docker compose restart`）通常也不需要重建，Volume 仍在。
- 以下场景建议重建：
        - 手动清空/删除了 `recipes` 集合。
        - 删除了相关 Docker Volume（例如 `docker compose down -v`）。
        - 变更了分块策略、Embedding 模型或向量维度。
        - 检索结果长期异常，且启动自检提示集合或索引不健康。
- 重建命令：

```bash
python -m gustobot.infrastructure.persistence.rebuild_milvus_index
```

### Neo4j 重复导入后数据变多
`neo4j_import.cypher` 会创建节点和关系。重复执行前先清空旧图谱：

```bash
docker compose exec neo4j cypher-shell "MATCH (n) DETACH DELETE n;"
docker compose exec neo4j cypher-shell -f /import/neo4j_import.cypher
```

### Redis JSON.SET 报错
确保 `docker-compose.yml` 中 Redis 使用 `redis/redis-stack-server:latest` 镜像。`redis:7-alpine` 缺少 RedisJSON 模块，LangGraph Checkpointer 需要该模块。

### LLM 调用超时
检查 `.env` 中 `LLM_BASE_URL` 是否正确、`LLM_API_KEY` 是否有效。默认使用阿里云百炼，需要有效的 DashScope API Key。

### 端口冲突
如果默认端口被占用，可在 `.env` 中修改：
```bash
PORT=8001      # FastAPI 端口
HOST=127.0.0.1 # 监听地址
```
